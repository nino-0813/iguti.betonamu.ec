# ベトナムECサイト構造設計書

**コンセプト**: ベトナムの手仕事を日本の暮らしへ  
**ターゲット**: 30〜50代女性（雑貨・文化・エシカル志向）  
**設計方針**: 日本のEC最適化 / ストーリー重視 / スマホファースト / SEO考慮 / シンプルで高級感

---

## 1. サイト全体構造（サイトマップ）

```
サイトマップ
├── トップページ (/)
├── カテゴリページ (/category/[category])
│   ├── 雑貨
│   ├── アパレル
│   └── バッグ
├── 商品ページ (/product/[id])
├── ストーリー (/story) ※ブランド・職人ストーリー
│   └── 職人紹介 (/story/[slug]) ※任意
├── ブログ (/blog)
│   └── 記事 (/blog/[slug])
├── FAQ (/faq)
├── 会社情報 (/about)
├── お問い合わせ (/contact)
│
├── [購入導線]
│   ├── カート (/cart)
│   ├── 注文完了 (/success)
│   └── 特定商取引法に基づく表記 (/tokushoho)
```

---

## 2. トップページ

### 目的
- **第一印象で「ベトナムの手仕事・ストーリー」を伝える**
- 商品閲覧・カテゴリへの導線を自然に配置
- 信頼感（実績・想い）を短時間で伝える

### 構成（上から順・スマホファースト）

| セクション | 内容 | 備考 |
|-----------|------|------|
| **ヒーロー** | キャッチコピー + コンセプト1行 + メインビジュアル（職人/商品の写真） | ファーストビューで「手仕事」「ベトナム」を一瞬で伝える |
| **ピックアップストーリー** | 今月の職人 or 特集記事 1本（タイトル・リード・リンク） | ストーリー入口。クリックでストーリー or ブログへ |
| **カテゴリナビ** | 雑貨 / アパレル / バッグ（アイコン or 画像 + ラベル） | タップでカテゴリページへ |
| **おすすめ商品** | 4〜8点（画像・商品名・価格・「詳しく見る」） | 新着 or 編集部ピックアップ。スクロールで続き |
| **なぜ私たちか（ミニAbout）** | 2〜3文 + 「ストーリーを見る」CTA | トップで会社の「想い」に触れさせる |
| **新着・ブログ** | 最新ブログ 2〜3件（タイトル・日付・リンク） | SEO・回遊・信頼 |
| **フッター** | カテゴリ / ストーリー / ブログ / FAQ / 会社情報 / お問い合わせ / 特定商取引法 | 全主要ページへのリンク |

### 必要コンテンツ
- ヒーロー用コピー（例：「ベトナムの手仕事を、日本の暮らしへ」）
- メインビジュアル画像（高解像度・縦横比統一）
- ピックアップ用ストーリー or ブログ1本（月1更新想定）
- おすすめ商品リスト（管理しやすいよう「ピックアップ」フラグ推奨）
- ミニAbout用 2〜3文（Aboutページと整合）

---

## 3. カテゴリページ

### 目的
- **「雑貨・アパレル・バッグ」ごとに商品を探しやすくする**
- カテゴリの世界観（素材・産地・職人）を短く伝える
- 一覧→商品詳細への導線を明確に

### 構成

| セクション | 内容 | 備考 |
|-----------|------|------|
| **カテゴリヘッダー** | カテゴリ名 + 1〜2行の説明文（このカテゴリの魅力・ストーリー） | SEO・共感。例：雑貨＝「暮らしに添える、一点一点手仕事の品」 |
| **絞り込み（オプション）** | 価格帯・素材・色など（初期はシンプルでOK） | 商品数が増えたら導入 |
| **商品グリッド** | カード型（画像・商品名・価格）2列（SP）/ 3〜4列（PC） | スクロールで続き読み or ページネーション |
| **カテゴリストーリー** | このカテゴリに関連する職人・産地の短い紹介 + ストーリーへのリンク | ストーリー重視の差別化 |
| **パンくず** | Home > カテゴリ名 | SEO・導線 |

### 必要コンテンツ
- 各カテゴリの説明文（雑貨・アパレル・バッグ）
- 商品データ（ID・名・価格・画像・カテゴリ）
- カテゴリ別に関連ストーリー or ブログを紐付け可能に

---

## 4. 商品ページ

### 目的
- **「欲しい」と思わせる = 見た目 + ストーリー + 信頼**
- 購入に必要な情報（価格・送料・在庫・仕様）を過不足なく表示
- カート投入までの導線を1〜2タップで

### 構成

| セクション | 内容 | 備考 |
|-----------|------|------|
| **商品画像** | メイン1枚 + サムネイル複数（スワイプ/クリックで切り替え） | スマホで拡大可能に。縦長推奨 |
| **基本情報** | 商品名・価格（税込表記）・送料・在庫表示 | 日本EC慣習に合わせる |
| **ストーリー** | 「この商品について」— 誰が・どこで・どう作ったか 2〜4文 | **差別化の核**。職人名・産地・こだわり |
| **仕様** | 素材・サイズ・重量・原産国・注意事項 | テキスト or 簡易表 |
| **数量・カートボタン** | 数量選択 + 「カートに入れる」CTA | 固定 or スクロールで追従（SP） |
| **関連ストーリー** | この商品の職人・産地のストーリー or ブログへのリンク | 深掘り用 |
| **よく一緒に買われる（任意）** | 2〜4点の関連商品 | 回遊・客単価 |
| **パンくず** | Home > カテゴリ > 商品名 | SEO |

### 必要コンテンツ
- 商品マスタ：名前・価格・在庫・画像・カテゴリ・仕様
- **商品ごとのストーリー文**（職人・産地・作り方の短い説明）
- 職人/工房と商品の紐付け（ストーリーページへのリンク用）

---

## 5. ブランドストーリーページ（ストーリー）

### 目的
- **「なぜベトナムの手仕事を届けるのか」を伝え、共感と信頼を得る**
- 職人・産地・こだわりを「読む体験」で届ける
- ストーリー経由で商品・カテゴリへ誘導

### 構成

| セクション | 内容 | 備考 |
|-----------|------|------|
| **ページタイトル** | 「ストーリー」+ サブコピー（例：つくり手と、私たちの想い） | 静かで上質なトーン |
| **ブランドストーリー** | なぜベトナムか・なぜ手仕事か・私たちの約束（3〜4ブロック） | 既存Aboutの内容を再構成可能 |
| **職人・工房紹介** | カード or リスト（写真・名前・短い紹介・「もっと読む」） | クリックで職人別ページ or ブログ記事へ |
| **この商品を見る** | 紹介した職人の商品一覧 or ピックアップ | ストーリー→購入の導線 |
| **ブログへの誘導** | 「もっと知る」→ ブログ一覧 or 特集記事 | 回遊 |

### 必要コンテンツ
- ブランドストーリー本文（既存Aboutを流用・強化）
- 職人・工房のプロフィール（名前・写真・紹介文・担当商品）
- 職人と商品の紐付けデータ

## 6. ブログ

### 目的
- **ストーリー・情報でリピートとSEOを強化**
- 職人インタビュー・現地レポート・使い方・コラムで「文化・手仕事」を伝える

### 構成

| ページ | 内容 |
|--------|------|
| **一覧 (/blog)** | 記事カード（サムネイル・タイトル・日付・リード）一覧。カテゴリ/タグ絞り込み（任意） |
| **記事 (/blog/[slug])** | タイトル・日付・本文（見出し・画像・引用）・関連商品・「ストーリーを見る」リンク・SNSシェア |

### 必要コンテンツ
- 記事：タイトル・slug・本文・サムネイル・公開日・カテゴリ
- 記事と商品・ストーリーの関連付け（任意）

## 7. FAQ

### 目的
- **購入前の不安を解消し、問い合わせを減らす**
- 送料・支払い・返品・配送日・ギフトなど、日本消費者が気にする項目を網羅

### 構成

| セクション | 内容 |
|-----------|------|
| **よくある質問** | アコーディオン or アンカーリンクで Q&A 一覧 |
| **カテゴリ** | 注文・支払い・送料・配送・返品・キャンセル・その他 |
| **お問い合わせ** | 「解決しない場合はお問い合わせ」CTA → /contact |

### 必要コンテンツ
- Q&A 一覧（問い・答を明文化）。運用で随時追加

## 8. 会社情報（About）

### 目的
- **法人・個人事業主としての信頼と透明性**
- 特定商取引法との役割分担（取引条件は tokushoho、想いは About）

### 構成

| セクション | 内容 |
|-----------|------|
| **店名・コンセプト** | 1〜2文で「何者か」を説明 |
| **私たちの想い** | なぜベトナムの手仕事を届けるか（ストーリーページへのリンク可） |
| **会社概要** | 屋号・代表・所在地・連絡先・設立（表形式でOK） |
| **リンク** | 特定商取引法に基づく表記・お問い合わせ |

### 必要コンテンツ
- 会社概要テキスト・代表名・住所・電話・メール等（法律に沿って記載）

## 9. お問い合わせ

### 目的
- **問い合わせを受け付け、返信までつなげる**
- フォームで項目を限定し、回答しやすくする

### 構成

| セクション | 内容 |
|-----------|------|
| **説明** | 返信目安・よくある質問へのリンク |
| **フォーム** | お名前・メール・お問い合わせ種別・本文・個人情報同意 |
| **送信後** | 送信完了メッセージ・自動返信メール（任意） |

### 必要コンテンツ
- フォーム項目定義・必須/任意・文字数制限
- 返信テンプレ・自動返信文（任意）

## 10. UI/UX 導線（全体方針）

- **スマホファースト**: 全ページをSPレイアウトで設計し、PCは余白・カラムで拡張。
- **ストーリーの露出**: トップ・商品・カテゴリのどこかで必ず「人・背景」に触れられるようにする。
- **購入導線の短さ**: トップ / カテゴリ / 商品 → カート → チェックアウト を 3〜4クリックで完了できるようにする。
- **信頼の積み重ね**: トップ（ミニAbout）→ ストーリー → 商品（ストーリー欄）→ FAQ → 会社情報 で一貫した「誠実さ」を伝える。
- **ナビ**: ヘッダーは ロゴ / カテゴリ（ドロップ or メニュー）/ ストーリー / ブログ / カート を常に見せる（SPはハンバーガー）。
- **フッター**: カテゴリ・ストーリー・ブログ・FAQ・会社情報・お問い合わせ・特定商取引法 をまとめて配置。

## 11. 購入までのユーザーフロー

```
[トップ / カテゴリ / ブログ・ストーリー]
        ↓
[商品一覧 or 商品詳細]
        ↓
[商品詳細でストーリーを読む] ← 信頼・共感
        ↓
[カートに入れる]
        ↓
[カートページで内容確認・数量変更]
        ↓
[レジに進む]（未ログインの場合はメール等のみでOKを想定）
        ↓
[配送先・支払い方法入力]
        ↓
[注文内容確認]
        ↓
[注文確定] → Stripe 等で決済
        ↓
[注文完了ページ (/success)] → メール案内
```

**ポイント**
- 会員登録を必須にしない（ゲストチェックアウト）で離脱を減らす。
- カート・レジの各段階で「送料」「支払い方法」「返品」へのリンクを置き、不安をその場で解消。
- 注文完了ページで「ストーリー・ブログ」「おすすめ商品」を少しだけ出し、回遊とリピートの種をまく。

## 12. SEO 構造の考慮

| 項目 | 対応 |
|------|------|
| **URL** | トップ / カテゴリ / 商品 / ストーリー / ブログ を階層化し、slug は短く・キーワード含む（例：/category/zakka, /product/abc-basket）。 |
| **タイトル** | 各ページで `ページ名 \| サイト名`。商品は「商品名 \| カテゴリ \| サイト名」など。 |
| **メタ説明** | 各ページ 120〜160 字で概要・ベネフィットを記載。 |
| **見出し** | H1 は1ページ1つ。H2→H3 の階層を守る。 |
| **構造化データ** | 商品ページに Product、組織に Organization、ブログに Article を検討。 |
| **内部リンク** | トップ→カテゴリ→商品、ストーリー→商品、ブログ→商品 を意図的に張る。 |

| **カテゴリ** | カテゴリ別探求・世界観 | ヘッダー説明 / 商品グリッド / カテゴリストーリー / パンくず | カテゴリ説明・商品データ・関連ストーリー |
| **商品** | 購入判断・ストーリーで差別化 | 画像 / 基本情報 / ストーリー / 仕様 / カートCTA / 関連・パンくず | 商品マスタ・商品別ストーリー・職人紐付け |

| **ブログ** | 情報・SEO・リピート | 一覧（カード）/ 記事（本文・関連商品） | 記事・サムネイル・カテゴリ |

| **会社情報** | 信頼・透明性 | 想い・会社概要・リンク | 概要・代表・住所・連絡先 |
| **お問い合わせ** | 受付・返信 | 説明・フォーム・送信後画面 | 項目定義・返信方針 |
